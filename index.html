Progress Snapshot

Current vertical slice

Match orchestration

  • Lobby -> TimerEnd -> StageLoading -> Curtain -> MatchLoop is active.
  • Random map loading pulls from ServerStorage/Maps into Workspace/CrossBoundActiveMap.
  • Role-based map spawn routing uses Killer and Survivor tags with SpawnLocation fallback.

Registry and relics

  • Registry selection is live and deterministic.
  • Tethered defaults to Will for solo iteration.
  • Relic progression is map-driven, server-authoritative, and supports multi-fix stabilization.

Seams and mutation

  • Relic mini-game supports pause, resume, chained rounds, and range-based interruption handling.
  • Seam readiness and mutation are live with debris and riser activation.
  • Clients receive smooth decaying screenshake on mutation.

Morphs and HUD

  • Morph pipeline supports idle, walk, run, injured, and burger-eat animation attributes plus runtime overrides.
  • Will survivor kit is active with Q speed boost and E burger heal.
  • HUD shows the round timer, map name, residue board, and placeholder Q/E ability UI.

Playtest Keys

Current vertical slice controls

Key map

Q

Trigger character action. Uses SurvivorAbilityIntent if Q is configured, otherwise sends the first Vestige intent.

E

Trigger character action. Uses SurvivorAbilityIntent if E is configured, otherwise communes with the nearest relic.

R

Attempt Breach at SeamAlpha. Tethered only.

T

Attempt Seam corruption. Wardens only.

Shift

Sprint with server-authoritative start and stop intents.

Ctrl

LeftCtrl or RightCtrl toggles shiftlock locally.

Z

Cycle character selection in The Registry for your assigned role.

Shift+X

Studio-only solo role toggle between Tethered and Wardens.

Test loop and role behavior

  1. Wait until Match Loop state appears in output.
  2. Stabilize relic fixes through the mini-game until the relic threshold is met.
  3. Confirm Seam readiness output and visual transitions.
  4. Press R to Breach and force The Verdict state.
  • With one player in server, the role is Tethered for objective flow tests.
  • With two or more players, one player is assigned Warden and the others are Tethered.
  • Vestige test input uses the current The Registry selection.
  • In Studio with one player, Shift+X lets you test both sides without opening a second client.

Studio Setup

Morphs, interactables, and folder placement

Morph setup

  • ServerStorage/CrossBoundMorphs/Tethered
  • ServerStorage/CrossBoundMorphs/Wardens
  • Optional fallback: Default inside each role folder
  • Models need a Humanoid and HumanoidRootPart
  • Spawn animation can come from an attribute or an Animation child

At Match Loop start, players morph into their selected Registry character. Changing selection with Z remorphs the active model. Default strategy is AppearanceOnly unless MorphStrategy is set to RigSwap.

Interactable setup

  • Relics use the Relic tag with optional RelicId, InteractRange, IsInteractable, PromptText, InputKey, RequiredFixes, and RelicLabel.
  • Every tagged Relic gets a clean white progress billboard that shows its own fix count.
  • Seams use the Seam tag with optional SeamId, InteractRange, IsInteractable, role-specific prompt and key attributes, and escape destination overrides.
  • Every tagged Seam gets a clean white progress billboard that shows total relics fixed.
  • StageMutationDebris and StageMutationRiser tags activate map mutation debris and risers after relic threshold completion.
  • Client controls use the nearest tagged interactable, and the interaction hint follows the mouse with a minimal white billboard style.

Studio placement

  • ReplicatedStorage/CrossBound
  • ServerScriptService/CrossBoundServer
  • StarterPlayer/StarterPlayerScripts/CrossBoundClient

Mirror these folders into Roblox Studio before testing so the server and client bootstrap scripts can find the shared modules and runtime hooks.
