Test loop

  1. Wait until Match Loop state appears in output.
  2. Press E three times to complete Relics.
  3. Confirm Seams opened output.
  4. Press R to Breach and force The Verdict state.

Role behavior

  • With one player in server, role is Tethered for objective flow tests.
  • With two or more players, one player is assigned Warden and others are Tethered.
  • Vestige test input uses the current The Registry character selection.
  • Use X in Studio with one player to simulate role swaps without launching a second client.

Studio Setup

Morphs, interactables, and folder placement

Morph setup

  • ServerStorage/CrossBoundMorphs/Tethered
  • ServerStorage/CrossBoundMorphs/Wardens
  • Optional fallback: Default under each role folder
  • Models need a Humanoid and HumanoidRootPart
  • Spawn animation can use an attribute or an Animation child

Morph runtime auto-ensures a Humanoid Animator and tries to clone Animate from the previous character. Default strategy is AppearanceOnly unless MorphStrategy is set to RigSwap.

Interactables

  • Relics use the Relic tag with optional RelicId, InteractRange, and IsInteractable.
  • Seams use the Seam tag with optional SeamId, InteractRange, and IsInteractable.
  • Client controls now use the nearest tagged interactable instead of hardcoded ids.

Studio placement

  • ReplicatedStorage/CrossBound
  • ServerScriptService/CrossBoundServer
  • StarterPlayer/StarterPlayerScripts/CrossBoundClient

Mirror the folder structure in Studio before testing so the server and client bootstrap scripts can find the shared modules and runtime hooks.
